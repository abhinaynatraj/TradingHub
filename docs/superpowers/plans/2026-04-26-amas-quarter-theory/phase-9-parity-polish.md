### Task 9.1: Replace SHA-256 hash with a portable polynomial hash on BOTH sides

The Phase 2 Python `canonical_hash` uses SHA-256[:8]. Pine has no built-in SHA. We replace both with a portable FNV-1a–style polynomial hash that both sides compute identically.

**Files:**
- Modify: `engine/state_vector.py:canonical_hash`
- Modify: `pine/quarter_theory.pine:canonicalHash`
- Modify: `tests/test_state_vector.py` (update expected hash values)

- [ ] **Step 1: Update Python `canonical_hash` to FNV-1a 64-bit**

In `engine/state_vector.py`, replace:

```python
def canonical_hash(key: str) -> str:
    """Return a base36-encoded 64-bit hash of the canonical key string."""
    digest = hashlib.sha256(key.encode("utf-8")).digest()
    n = int.from_bytes(digest[:8], "big", signed=False)
    return _to_base36(n)
```

With:

```python
def canonical_hash(key: str) -> str:
    """FNV-1a 64-bit hash of UTF-8 bytes, base36-encoded.

    Pine implements the same algorithm in its canonicalHash() — they MUST
    produce byte-identical output for the same input string. Tested in
    test_state_vector.py::test_pine_parity once Phase 9 wiring is complete.
    """
    h = 0xcbf29ce484222325       # FNV offset basis
    prime = 0x100000001b3
    mask = 0xFFFFFFFFFFFFFFFF
    for byte in key.encode("utf-8"):
        h ^= byte
        h = (h * prime) & mask
    return _to_base36(h)
```

- [ ] **Step 4: Replace Pine `canonicalHash` with FNV-1a 64-bit equivalent**

In `pine/quarter_theory.pine`, replace the placeholder `canonicalHash` with:

```pinescript
// FNV-1a 64-bit hash → base36. Mirrors engine/state_vector.py:canonical_hash.
canonicalHash(string s) =>
    int h = 0xcbf29ce484222325
    int prime = 0x100000001b3
    int mask = 0x7FFFFFFFFFFFFFFF  // Pine ints are signed 64-bit; use 63-bit mask
    int len = str.length(s)
    for i = 0 to len - 1
        // Get codepoint of char at i. UTF-8 of ASCII = single byte equal to codepoint.
        // For non-ASCII chars (rare in our state keys — only ASCII used), this needs
        // multi-byte expansion; since our schema is ASCII-only, single-codepoint is fine.
        c = str.charat(s, i)
        // Pine's str.charat returns the unicode codepoint as an int (in v6).
        // If c > 127, would need UTF-8 multi-byte. Our state keys never contain such.
        h := (h ^ c) * prime
        h := h & mask
    // base36
    string digits = "0123456789abcdefghijklmnopqrstuvwxyz"
    string out = ""
    int n = h
    while n > 0
        d = n % 36
        out := str.substring(digits, d, d + 1) + out
        n := n / 36
    out == "" ? "0" : out
```

> **Note on int width:** Python uses a 64-bit unsigned hash; Pine ints are 64-bit signed. We use a 63-bit mask in Pine so the result fits in positive int range. Update Python to match: also use the 63-bit mask. Edit `engine/state_vector.py`:

```python
    mask = 0x7FFFFFFFFFFFFFFF  # match Pine 63-bit positive int range
```

- [ ] **Step 2: Add a Python test that generates the parity fixture file and a test that consumes the user-pasted Pine output**

File: `tests/test_pine_parity.py`

```python
"""Pine ↔ Python parity test.

Workflow:
1. Run engine/build.py → writes tests/fixtures/state_cases.json with N keys
   and their Python-computed hashes.
2. User opens pine/_parity_runner.pine in TradingView, ensures the testCases
   array matches state_cases.json["pairs"][*]["key"], adds to chart.
3. User copies the multi-line label output and saves it as
   tests/fixtures/pine_parity_export.json.
4. This test loads both and asserts the hashes agree.
"""
from __future__ import annotations

import json
from pathlib import Path

import pytest

from engine.state_vector import canonical_hash


HERE = Path(__file__).resolve().parent
FIXTURES = HERE / "fixtures"


def test_canonical_hash_matches_state_cases_fixture():
    """Every key in state_cases.json hashes to the value recorded in the fixture."""
    p = FIXTURES / "state_cases.json"
    if not p.exists():
        pytest.skip("state_cases.json not generated yet — run engine/build.py")
    data = json.loads(p.read_text())
    for pair in data["pairs"]:
        actual = canonical_hash(pair["key"])
        assert actual == pair["hash"], (
            f"Python hash mismatch for key {pair['key']!r}: "
            f"expected {pair['hash']}, got {actual}"
        )


def test_pine_parity():
    """Pine-side hashes (from pine_parity_export.json) match Python-side hashes."""
    cases_path = FIXTURES / "state_cases.json"
    pine_path = FIXTURES / "pine_parity_export.json"
    if not cases_path.exists():
        pytest.skip("state_cases.json not generated — run engine/build.py")
    if not pine_path.exists():
        pytest.skip(
            "pine_parity_export.json not present. Manual workflow:\n"
            "  1. Open pine/_parity_runner.pine in TradingView\n"
            "  2. Confirm the testCases array matches keys in state_cases.json\n"
            "  3. Add to chart, copy the output label's text content\n"
            "  4. Save as tests/fixtures/pine_parity_export.json"
        )

    py_data = {p["key"]: p["hash"] for p in json.loads(cases_path.read_text())["pairs"]}
    pine_data = {p["key"]: p["hash"] for p in json.loads(pine_path.read_text())["pairs"]}

    common_keys = set(py_data.keys()) & set(pine_data.keys())
    assert len(common_keys) > 0, "no common keys between Python and Pine fixtures"
    for key in sorted(common_keys):
        assert py_data[key] == pine_data[key], (
            f"PARITY VIOLATION for {key!r}:\n"
            f"  Python: {py_data[key]}\n"
            f"  Pine:   {pine_data[key]}"
        )
```

```python
def build_for_symbol(sym: str, start: str | None = None, end: str | None = None) -> tuple[str, pd.DataFrame]:
    """Build empirical + qpair Pine snippets for one symbol.

    table = C.TABLE_FOR_SYMBOL[sym]
```

```python
def emit_parity_fixture(empirical_df: pd.DataFrame, out_path: Path, n_cases: int = 30) -> None:
    """Emit a sample of state keys + Python-side hashes for Pine parity validation."""
    import json
    from engine.state_vector import canonical_hash
    keys = empirical_df["state_key"].drop_duplicates().head(n_cases).tolist()
    pairs = [{"key": k, "hash": canonical_hash(k)} for k in keys]
    out_path.parent.mkdir(parents=True, exist_ok=True)
    out_path.write_text(json.dumps({"pairs": pairs}, indent=2) + "\n")
```

```markdown
# Amas Quarter Theory

Live TradingView Pine v6 indicator that overlays bootcamp price-action concepts (15m quarter analysis, 1h doji/line, 3h triad line/apex, 05-box bands, midline mechanics) on the price chart, with empirical probability readouts pre-computed from historical NQ/ES 1m data.

![screenshot](assets/screenshot.png)

## What it does

- **Quarter dividers** at :15, :30, :45 of every hour (NY tz).
- **In-stat / out-of-stat extreme labels** anchored to the candle that printed the high/low.
- **Sweeper markers** + **bias-shift arrows** + **doji-confirmed** flag.
- **Hour & triad close verdicts** (line-up / line-down / doji-hour; line / apex / doji triad).
- **Apex-hour highlight** on C2 of an apex triad.
- **05 box** (full-height vertical column over :00–:04 of each hour).
- **±0.05% / ±0.10% bands** anchored to the box edges, drawn through that hour only.
- **Band rejection markers** (wick-and-close).
- **1h midline + 3h midline**, with prior-vs-current source resolution per the inside-bar rule.
- **Midline reaction markers** (support / reject).
- **Live probability readouts** centered under each 1h block and 3h triad, with sample count, confidence color, and quarter-pair recommendation.
- **3h running OHLC + flip-level overlay**.
- **Tooltips on hover** showing full state breakdown.
- **Alerts** (sweeper, doji-confirmed, apex-confirmed, midline reaction, band rejection).
- **Dark / Light / Custom** color palettes.
```
