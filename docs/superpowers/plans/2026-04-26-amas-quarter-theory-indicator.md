# Amas Quarter Theory — Implementation Plan (Index)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a Pine v6 TradingView indicator that applies bootcamp price-action concepts (15m quarters, 1h doji/line, 3h triad line/apex, 05-box bands, midlines) on the chart with empirical probability readouts pre-computed from `nq_1m` / `es_1m` historical data.

**Spec:** [`docs/superpowers/specs/2026-04-26-amas-quarter-theory-indicator-design.md`](../specs/2026-04-26-amas-quarter-theory-indicator-design.md)

**Architecture:** Single Pine indicator (`pine/quarter_theory.pine`) embedding probability tables generated by an offline Python engine (`engine/`). Engine pattern mirrors `Amas Models/`: pure-function classifier, deterministic build, parquet caches, byte-reproducible Pine output. Pine ↔ Python state-key parity is the load-bearing correctness invariant.

**Tech Stack:** Pine v6 · Python 3.14 · DuckDB 1.4.4 · pandas · pytest. Reuses Amas Models patterns (`stats.wilson_ci`, `db.py`, `OUTCOME_MAX_BARS=1440`, same-bar tie → SL).

## Cross-phase conventions

**Repo location:** All work lives under `Statistic.ally/Amas Quarter Theory/`. Engine scripts self-locate; run from that folder unless otherwise noted.

**Working directory in commands:** Plans assume CWD = `Statistic.ally/Amas Quarter Theory/` unless explicit absolute paths are given.

**DB:** Read-only access to `../Fractal Sweep/candle_science.duckdb`. Tables `nq_1m`, `es_1m`. Always convert to NY tz at SQL layer (`timezone('America/New_York', timestamp)`).

**Schema version:** `SCHEMA_VERSION = "v1"` — single constant in both Python (`engine/state_vector.py`) and Pine (`quarter_theory.pine`). Changing the state-vector schema requires bumping this and re-running parity.

**Test framework:** pytest. Run from `Amas Quarter Theory/` folder: `python3 -m pytest tests/ -q`.

**Commit cadence:** Commit after each task's tests pass. Commit messages: `feat:` / `test:` / `fix:` / `chore:`.

**TDD:** Every step pair is `test first → run (FAIL) → minimal impl → run (PASS) → commit`. No code before its failing test exists.

**No placeholders:** Every step has the actual code or command to run. Steps that say "implement X similar to Y" are bugs in the plan — flag them.

## Correctness invariants (apply to every phase)

These mirror `Amas Models/CLAUDE.md`. Tests enforce all of them — not gated on DEBUG.

- **A. Causal sampling** — state-vector at time T uses only bars `ts ≤ T`.
- **B. Pine ↔ Python parity** — state-key strings byte-identical.
- **C. Classifier parity** — hour/triad classification byte-identical.
- **D. Sweep detection** — strict break, equality does not trigger.
- **E. 05-box arithmetic** — minute bars at :00,:01,:02,:03,:04 inclusive; bands at exact `box × 1.0005`, `box × 1.0010`, `box × 0.9995`, `box × 0.9990`.
- **F. Midline source** — strict on both sides; equality counts as broken-out.
- **G. Quarter-pair backtest fidelity** — same-bar tie → SL, `OUTCOME_MAX_BARS=1440`, expired excluded from WR/EV, dedup by `(triad_id, pair, decision_point)`.
- **H. Wilson CI on every WR / probability**; `n` always visible.
- **I. Determinism** — `engine/build.py` byte-reproducible.
- **J. Source-size budget** — `_generated_tables.pine` ≤ 900 KB, asserted in `build.py`.
- **K. Fallback monotonicity** — strip order never regresses.
- **L. Data-quality on every load** — monotonic, unique, OHLC-sane, schema match.

## Decision points (full list, used across phases)

These are the times the empirical engine samples state and Pine re-evaluates probabilities:

1. Quarter close (every :15, :30, :45, :00)
2. Sweep event mid-quarter
3. Midline reaction confirmed
4. Band rejection confirmed
5. Hour close (:00)
6. Triad close (every 3 hours at block boundary)
7. 05-box close (at :04 of every hour)

Phase 3 implements the offline sampler; Phase 6 implements the live evaluator; both must agree on what counts as each event.
